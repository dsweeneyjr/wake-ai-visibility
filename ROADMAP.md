# AI Search Intelligence Roadmap

The goal of this project is to evaluate how Wake Tech appears in AI-powered search experiences and determine the best approach for incorporating AI visibility into ongoing website monitoring and optimization.

## Current Prototype

The current prototype demonstrates a complete monitoring workflow:

- Maintain a library of prospective-student search prompts
- Evaluate visibility across ChatGPT, Gemini and Perplexity
- Track whether Wake Tech is mentioned
- Record approximate recommendation position
- Identify competitors appearing in AI responses
- Score visibility performance
- Compare results across scans
- Track visibility trends over time
- Identify weak categories and missed mentions
- Generate prioritized content recommendations

The current automated scan uses simulated response data. This allows the team to develop and evaluate the monitoring methodology, scoring model, reporting interface and recommendation workflow before committing to paid APIs or third-party platforms.

## Next Phase: Live AI Monitoring

The next phase would replace simulated scan results with live responses from AI platforms and search experiences.

Potential monitoring sources include:

- OpenAI models and ChatGPT-related search experiences
- Google Gemini
- Google AI Overviews
- Perplexity
- Other emerging AI search and answer platforms

Live monitoring would allow Wake Tech to repeatedly test real prospective-student questions and measure changes in visibility over time.

## Monitoring Workflow

A production workflow could:

1. Run a defined library of high-value student questions on a scheduled basis.
2. Capture responses from supported AI platforms.
3. Detect whether Wake Tech is mentioned and how prominently it appears.
4. Identify competitors and cited sources.
5. Score results by platform, topic and prompt.
6. Compare results with previous scans.
7. Flag meaningful gains, declines and missed mentions.
8. Generate content optimization recommendations for review.

## Estimated Operating Cost

A focused internal monitoring system should be relatively inexpensive to operate compared with enterprise AI visibility platforms.

For a modest prompt library and scheduled scans, estimated API costs could initially be approximately **$25–$100 per month**, depending on:

- Number of prompts monitored
- Number of platforms queried
- Scan frequency
- Model selection
- Response length and token usage

Actual costs would be validated during live API testing before production implementation.

## What Success Looks Like

The objective is not simply to produce a visibility score.

The system should help the web and marketing teams answer practical questions:

- Where is Wake Tech being recommended?
- Where is Wake Tech missing?
- Which competitors are appearing instead?
- Which programs and topics have the strongest visibility?
- Which pages or content areas should be improved first?
- Are optimization efforts improving AI visibility over time?

## Recommended Path Forward

### Phase 1 — Prototype and Evaluation
**Status: In progress**

Develop the monitoring methodology, prompt library, scoring model, dashboard, trend reporting and recommendation workflow.

### Phase 2 — Limited Live Testing

Connect selected live AI data sources and test a controlled set of high-value prompts. Compare results with the prototype scoring methodology and validate operating costs.

### Phase 3 — Ongoing Monitoring

Establish a recurring scan schedule and incorporate AI visibility reporting into existing website analytics, SEO and content optimization workflows.

### Phase 4 — Optimization and Measurement

Use monitoring results to prioritize content improvements, measure changes over time and refine Wake Tech content for stronger visibility in both traditional and AI-powered search experiences.

This project is currently an evaluation prototype. Its purpose is to help Wake Tech determine the most effective, sustainable and measurable approach to AI search visibility monitoring before making a larger technology investment.
